PROJECT OVERVIEW
================================================================================

A standalone Python GUI application that replicates the functionality of the
Emacs strkm-mode "process and export XLS" operation. The application:

1. Accepts Starkmann email files (.txt, .eml) as input
2. Parses bibliographic book data from email format
3. Extracts structured information (authors, title, publisher, etc.)
4. Exports to Excel (XLSX) format with 10 columns
5. Generates accompanying .books file for data preservation
6. Provides user-friendly GUI with error handling

COMPATIBLE PLATFORMS:
- Windows (XP+) with Python 3.7.2+
- macOS with Python 3.7+
- Linux (Ubuntu/Debian/Fedora) with Python 3.7+

USAGE EXAMPLE
================================================================================

1. User runs: python starkmann_email_processor.py

2. GUI window opens with:
   Input File:  [Browse...]
   Output File: [Browse...]
   [Process & Export]

3. User clicks "Browse..." and selects: "25-Nov-24.txt"
   System auto-generates output: "2024_11_25.xlsx"

4. User clicks "Process & Export"

5. System:
   - Reads email file (windows-1252 encoding)
   - Finds separator lines: ================================================================(XXXX)
   - Extracts title, authors, publisher, ISBN, price for each block
   - Creates XLSX file with 283 rows + 1 header row
   - Saves .books file with raw data
   - Shows success message with file locations

6. User finds two files in output directory:
   - 2024_11_25.xlsx (26 KB, opens in Excel/LibreOffice)
   - 2024_11_25.books (228 KB, raw data for archival)
